# __APP_TITLE__

Framework-neutral starter for a new OrganizationLaunchpad app.

This starter uses:

- plain TypeScript
- Vite
- the shared auth shell in `shared/auth`
- portable login, signup, callback, and guard UI through Web Components

## What It Gives You

- `/login`
- `/signup`
- `/auth/callback`
- `/app` as a protected route

## Why This Starter Exists

This is the default “new app” path when you want shared auth without inheriting any Svelte-specific app code from `apps/web`.
