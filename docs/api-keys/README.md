# API Keys

Use this directory to document every external service an app depends on.

Each service entry should capture:

- service name
- purpose
- signup or console URL
- steps to obtain the key
- required scopes or permissions
- environment variable names
- local vs production ownership
- rotation notes

Suggested template:

```md
## Service name

Purpose:

Console URL:

How to get the key:

Environment variables:

Ownership:

Rotation notes:
```

This documentation is intentionally part of the scaffold because operational setup is one of the main things vibecoded apps usually omit.
